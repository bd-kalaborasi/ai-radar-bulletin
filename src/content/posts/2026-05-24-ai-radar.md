### OpenAI Codex Goal Mode reaches general availability across app, IDE, and CLI

**Source:** https://developers.openai.com/codex/changelog · OpenAI Codex Changelog · 2026-05-21
**Verification:** T2 verified · changelog · dev-tools / workflow-automation

OpenAI graduated Goal Mode to general availability on 21 May 2026 (version 26.519), available in the Codex app, VS Code and JetBrains extensions, and CLI. Goal Mode lets developers assign a broad objective — such as raising test coverage in a specific module — and Codex plans, executes, self-checks, and course-corrects across session breaks over hours or days without further prompting. The same release shipped Appshots for macOS (pressing both Command keys attaches a screenshot of the foreground app window to a Codex thread) and remote computer use on locked Macs, including from Codex Mobile, with short-lived authorization and screen-cover safeguards.

**Why it matters for automation/productivity:** Goal Mode shifts the developer usage pattern from per-prompt interactions to objective-level delegation — teams can assign multi-session jobs and return to results rather than supervising each turn. Locked-Mac computer use extends coverage to overnight and unattended automation workflows.

**Key claims:**
- Goal Mode available in app, VS Code/JetBrains extension, and CLI → Codex changelog primary (2026-05-21)
- Appshots: both Command keys attaches foreground app window → Codex changelog primary
- Remote computer use on locked Mac with short-lived authorization safeguards → Codex changelog primary

- https://aiweekly.co/alerts/openai-codex-launches-appshots-and-stable-goal-mode (T3, corroborating)

### Walmart Sparky Q1 2026: weekly active users double, average order value 35% higher

**Source:** https://www.digitalcommerce360.com/2026/05/22/walmart-sparky-agent-ai-sales-supply-chain/ · Digital Commerce 360 · 2026-05-22
**Verification:** T2 secondary · earnings call report · workflow-automation / ai-for-business
[Primary: Q1 FY2026 earnings call, Walmart CEO John Furner, 22 May 2026; confirmed across multiple retail trade outlets]

Walmart CEO John Furner stated on the 22 May Q1 2026 earnings call that weekly active users of the Sparky shopping agent more than doubled quarter-over-quarter, and that Sparky users show an average order value approximately 35% higher than non-Sparky customers. Units purchased through Sparky more than quadrupled since the prior fiscal quarter. The company added Spanish-language support, in-store access, and automatic reordering for repeat items during the period. Sparky is accessible within Walmart's own app and website, as well as via ChatGPT and Google Gemini integrations launched in 2026.

**Why it matters for automation/productivity:** These are among the first public commercial-scale metrics for a deployed shopping agent, providing a concrete reference for what agentic AI can move at retail scale. Teams building consumer or B2B commerce agents can use Walmart's 35% average order value lift as a benchmark — with the caveat that no attribution methodology was disclosed.

**Key claims:**
- Weekly active users: more than doubled QoQ → Walmart Q1 earnings call (secondary via Digital Commerce 360)
- Average order value: ~35% higher for Sparky users vs non-users → Walmart Q1 earnings call (secondary)
- Units: more than quadrupled since previous fiscal quarter → Walmart Q1 earnings call (secondary)

**Cross-references:**

### Salesforce Agentforce Coworker launches in beta inside every Salesforce search bar

**Source:** https://developer.salesforce.com/docs/data/agentforce-coworker/guide/agentforce-coworker-turn-on-infrastructure.html · Salesforce Developer Docs · 2026-05-21
**Verification:** T2 secondary · announcement · workflow-automation / productivity-ai
[Salesforce developer documentation confirmed; CEO Marc Benioff announced availability via social post 21 May 2026; detailed coverage published 22 May via Salesforce Ben]

Salesforce launched Agentforce Coworker in beta on 21 May 2026, embedding an AI assistant into the search bar across Salesforce products, Slack, and Microsoft Teams. Coworker accesses CRM data, customer history, opportunities, and cases in real time and can execute actions against those records — not just retrieve. Available today for all Agentforce customers on Enterprise, Unlimited, and Agentforce 1 editions; administrators need the Access_Ai_Search Permission Set Group to activate the feature. No additional pricing beyond existing Agentforce licenses was announced.

**Why it matters for automation/productivity:** Placing an AI assistant at the search layer of Salesforce means sales, service, and ops teams can query and act on CRM data without navigating between modules. Early testers report resolving complex sales and ERP data queries that previously required 45–60 minutes of manual work. Beta quality caveats apply.

**Key claims:**
- Available: Agentforce Enterprise, Unlimited, Agentforce 1 editions → Salesforce Ben citing Benioff announcement (T3)

- Accessible in Salesforce, Slack, and Microsoft Teams → Salesforce Ben (T3)

### TeamPCP supply chain worm hits GitHub internals and AI company SDKs

**Source:** https://therecord.media/github-confirms-teampcp-hack-customers-unaffected · The Record from Recorded Future News · 2026-05-20
**Verification:** T2 verified · security incident report · dev-tools / policy-regulation

GitHub confirmed that a poisoned version of the Nx Console VS Code extension (version 18.95.0, published 18 May 2026) gave threat group TeamPCP access to internal repositories. Attackers exfiltrated approximately 3,800 internal GitHub repos via harvested credentials; GitHub stated customer data was unaffected and the breach was detected and contained. As part of the broader Shai-Hulud campaign active since September 2025, TeamPCP also trojanized npm and PyPI SDKs: OpenAI confirmed two employee devices were compromised with limited credential material exfiltrated from a subset of internal source code repositories; Mistral AI confirmed its SDKs were trojaned and advertised for sale on a cybercrime forum. CVE-2026-45321 (CVSS 9.6) describes the TanStack CI/CD chain exploitation used to spread the worm across 170+ packages.

**Why it matters for automation/productivity:** Any development team using TanStack router ecosystem packages, Mistral AI SDKs, or OpenAI SDKs installed during the affected window should audit their dependency tree and rotate credentials. Developer environments running AI coding agents — including Claude Code, Codex, and Cursor — are specifically targeted by TeamPCP's persistence hooks for IDEs.

**Key claims:**
- ~3,800 internal GitHub repos exfiltrated → GitHub investigation statement (T2)
- Nx Console v18.95.0 poisoned, published 18 May 2026 → Help Net Security (T2)
- 170+ npm packages affected → phoenix.security technical analysis (T2-T3)
- CVE-2026-45321, CVSS 9.6 → Tenable FAQ (T2)
- OpenAI: two employee devices compromised, limited credential exfiltration → OpenAI statement (T2)
- Mistral AI: SDKs trojaned → Mistral confirmation cited in T2 reporting

**Cross-references:**
